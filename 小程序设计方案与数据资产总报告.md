## 二、数据资产详细说明

### 2.1 知识点与公式库 (`data_knowledge_base.json`)

**公式**（112个）：每个包含 id、name、LaTeX表达式、description、chapter、topic、variables

**知识点**（140个）：每个包含 id、title、content、chapter、topic、related_formulas

**覆盖范围**：
- **几何光学**：反射折射定律、全反射、费马原理、近轴光学、球面成像、放大率、拉赫不变量、理想光学系统（高斯/牛顿公式）、光组组合、透镜、平面镜/棱镜/光楔、光阑/景深/远心光路、像差（球差/彗差/像散/场曲/畸变/色差）、典型系统（眼睛/放大镜/显微镜/望远镜/摄影）
- **物理光学**：麦克斯韦方程组、波动方程、光速、干涉（杨氏双缝/薄膜/牛顿环/迈克尔逊/F-P/多光束）、衍射（菲涅尔/夫琅和费/单缝/圆孔/矩孔/多缝/光栅）、偏振（双折射/波片/旋光/法拉第效应）

### 2.2 选择题题库 (`data_choice_questions.json`)

**70道题**，每题包含：
- `question`：题目文本
- `options`：4个选项（A/B/C/D）
- `answer`：正确答案
- `explanation`：详细解析
- `chapter/topic/difficulty`：分类标签
- `knowledge_tags`：关联知识点id
- `wrong_hint`：答错时弹出的提示（区分概念错误/公式混淆/符号规则）

**错误提示设计示例**（以全反射题为例）：
```json
"wrong_hint": {
  "type": "概念错误",
  "message": "全反射必须同时满足两个条件：方向是从光密到光疏，且角度要超过临界角。仅满足一个条件不够。"
}
```

### 2.3 计算题步骤填空 (`data_calculation_steps.json`)

**通用数据结构（Schema）**：
```
question
  ├── id, title, topic, difficulty
  ├── problem_statement（题目描述）
  ├── given_conditions[]（已知条件：符号、数值、单位）
  ├── asked（求解目标）
  ├── steps[]（步骤列表）
  │     ├── step_id, step_name, input_type（formula/numerical/choice）
  │     ├── expected_input（正确答案/模式/容差）
  │     ├── validation_rules（验证规则：精确/模糊/公式等价）
  │     ├── common_errors[]（常见错误诊断）
  │     │     ├── error_type（sign_error/concept_error/formula_error...）
  │     │     ├── diagnosis_message（诊断信息）
  │     │     ├── root_cause（根本原因）
  │     │     └── remediation_hint（纠正提示）
  │     └── scaffolding（辅助推送：连续错误N次后提示）
  └── error_diagnosis_map（全局错误诊断映射）
```

## 三、小程序页面结构设计

### 3.1 首页（Tab 1）：公式手册
- **顶部**：搜索栏（按公式名/关键词搜索）
- **中部**：思维导图（可展开/折叠，点击节点进入公式详情页）
- **底部**：快捷入口（高频公式/易混淆对比/章节速查）
- **公式详情页**：LaTeX渲染 + 变量说明 + 关联知识点 + 收藏按钮

### 3.2 刷题（Tab 2）：题目训练
- **子页面1：选择题**
  - 题目展示（支持LaTeX公式渲染）
  - 选择答案 → 即时判断 → 正确：下一题；错误：弹出 `wrong_hint` 记忆卡
  - 收藏错题、查看解析
- **子页面2：计算题步骤填空**
  - 题目展示（已知条件列表化）
  - 步骤1输入 → 提交 → 系统验证 → 正确进入步骤2；错误弹出诊断
  - 诊断信息区分：符号错误/公式错误/概念错误/计算错误
  - 连续N次错误后推送辅助材料（公式参考/微课提示/类似例题）
- **子页面3：知识点卡片**
  - 翻转卡片：正面知识点名称 → 背面内容/公式
  - 支持填空模式（遮住公式一部分，用户填写）
  - 按章节分类浏览

### 3.3 错题本（Tab 3）：个人数据
- 错题列表（按错误类型分类：概念错误/公式混淆/符号规则）
- 薄弱点报告（"你在'符号规则'上错了8次，建议复习第1章符号约定"）
- 收藏的题目和公式
- 学习进度（各章节刷题覆盖率）

### 3.4 我的（Tab 4）：设置
- 难度偏好
- 每日刷题目标
- 清除缓存/导出数据

## 四、关于画图题的技术结论

### 4.1 调研结论：不推荐多模态AI

根据 `research_multimodal_ai.md` 调研：

| 维度 | 多模态AI API | 纯规则生成（Canvas+规则） |
|------|-------------|------------------------|
| **准确率** | 25-35%（视觉几何推理） | 100%（确定性规则） |
| **延迟** | 1-5秒（需网络+后端转发） | 毫秒级（本地实时） |
| **成本** | 持续按量计费（~0.01-0.1元/次） | 几乎为零 |
| **离线可用** | 否 | 是 |
| **步骤级纠错** | 困难（黑盒，难以定位具体步骤） | 精确（白盒，每个步骤独立验证） |

> **关键依据**：RBench-V（2025）测试显示，最强模型o3在视觉几何推理上仅25.8%准确率，人类为82.3%。光学光路图的核心判断点（光线是否平行、是否过焦点、交点位置）恰好是多模态AI的系统性短板。

### 4.2 推荐方案：纯规则生成

当后续扩展画图题时，采用以下方案：

- **前端**：微信小程序 Canvas 2D，用户用手指画光线
- **后端/本地**：几何规则引擎验证
  - 判断光线斜率（是否平行于光轴）
  - 判断光线是否经过特定点（如F'）
  - 判断两条光线的交点位置
  - 步骤级验证：每画一条线就即时判断
- **数据基础**：已有画图合集提取的18张图片，包含6类标准题型规则

**结论**：当前阶段（A+B）不涉及画图题；后续扩展画图题时，无需多模态AI，直接用规则引擎即可。

## 五、数据文件的关联关系

```
data_formula_mindmap.json
    ├── formula_id → 关联 → data_knowledge_base.json[formulas]
    └── knowledge_id → 关联 → data_knowledge_base.json[knowledge]

data_choice_questions.json
    └── knowledge_tags → 关联 → data_knowledge_base.json[knowledge]
    └── wrong_hint → 引用 → data_knowledge_base.json[formulas/knowledge]

data_calculation_steps.json
    ├── steps[].formula_reference → 关联 → data_knowledge_base.json[formulas]
    ├── common_errors[].knowledge_ref → 关联 → data_knowledge_base.json[knowledge]
    └── scaffolding.formula_reference → 关联 → data_knowledge_base.json[formulas]
```

**统一数据层**：所有文件通过 `formula_id` 和 `knowledge_id` 建立关联，前端可以按需加载。

## 六、技术实现建议

### 6.1 数据层（已就绪）
- 5个JSON文件作为静态数据源
- 建议部署到云数据库（如微信云开发数据库）或打包到小程序本地
- 公式LaTeX渲染：使用 `miniprogram-latex` 或 `katex` 小程序适配版

### 6.2 业务层（待开发）
- 选择题验证：直接比对答案字符串
- 计算题步骤验证：
  - 数值型：比对数值+容差（如 `abs(user_input - expected) < tolerance`）
  - 公式型：使用简化公式等价检查（如转换为标准形式后比对）
  - 符号型：检查正负号
- 错误诊断：根据 `common_errors` 中的 `error_pattern` 匹配用户输入，返回对应 `diagnosis_message`
- 薄弱点统计：记录用户错误类型分布，生成报告

### 6.3 表现层（待开发）
- 思维导图：使用递归组件渲染树形结构
- 公式渲染：KaTeX 渲染 LaTeX 字符串
- 答题交互：步骤填空使用输入框+即时验证动画
- 答错记忆卡：底部弹出卡片，包含提示信息+关联公式+"去复习"按钮
